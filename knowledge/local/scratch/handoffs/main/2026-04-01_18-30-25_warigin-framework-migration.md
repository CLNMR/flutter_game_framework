## Learnings

### Extension methods cannot override instance methods
- Dart extensions are statically dispatched based on the receiver's static type
- Inside an extension on `Game`, calling `customStartLogic()` resolves to the extension defined for `Game`, NOT a subclass's extension — even at runtime
- This was a latent bug in TB's customStartLogic (never called by framework's startGame)
- Solution: convert extension methods to virtual instance methods on the abstract class

### `late final` fields break with json_serializable round-trips
- `late final Map remainingCards` gets set in the constructor body
- When `fromJson` deserializes, json_serializable tries to set it again → `LateInitializationError: already initialized`
- Fix: use `late` (not `late final`) for fields initialized in constructor body AND deserialized

### Duplicate Riverpod providers cause silent null values
- Warigin had its own `authStateProvider` and `userProvider` that shadowed the framework's
- The framework's login/auth wrote to framework providers, but Warigin's `ref.user` read from Warigin's (empty) providers → always null
- Fix: delete duplicate providers, delegate to framework via explicit extension delegation

### `getPlayerIndex` returning -1 cascades into multiple crashes
- Many widgets call `game.getPlayer(ref.user)` which calls `getPlayerIndex` → `players[-1]` → RangeError
- Root cause was `ref.user` being null (see above), but the pattern is fragile
- Multiple callers need guards: `_userPlayer` getter in in_game_display, `removeLast()` guard in player_instructions_row

### GameState dual-layer pattern
- Framework's `GameState`: waitingForPlayers, running, finished, abandoned (top-level lifecycle)
- `WGGameState`: notRunning, bidding, cardSelection, heroSelection, inProgress (game-specific sub-states)
- `gameState` (inherited) tracks lifecycle; `wgGameState` (WGGame field) tracks game phase
- Game screen uses two-level switch: first on `gameState`, then on `wgGameState`
- Setting both when transitioning: `gameState = GameState.running; wgGameState = WGGameState.bidding;`

### `Map.from()` creates shallow copy — shared list references
- `Map<int, List<CardCatalog>>.from(initialCards)` copies map entries but NOT the lists
- `remainingCards[3]` and `initialCards[3]` are the same List object
- Pre-existing issue, not caused by migration

### google_fonts 6.x incompatible with Flutter 3.38 web
- Uses legacy `AssetManifest.json` which newer Flutter doesn't serve
- Upgrade to ^8.0.0 fixes it (uses built-in AssetManifest API)

### `pubspec_overrides.yaml` takes precedence over `pubspec.yaml` dependency_overrides
- `dio_web_adapter ^2.1.0` override had to go in `pubspec_overrides.yaml`, not `pubspec.yaml`
- The overrides file's `dependency_overrides` section wins
